\begin{abstract}
KV cache dominates GPU memory consumption in LLM inference, becoming the core bottleneck for long-context serving. Existing KV cache compression methods either (i) select a subset of tokens and discard the rest entirely, or (ii) apply uniform precision reduction across all tokens. Both paradigms ignore the per-head, per-block heterogeneity in attention structures.

This paper presents ACCORD-KV, a decentralized KV cache reorganization framework grounded in Attention Contracts: semantic descriptors that encode minimum precision requirements per KV block. We make a critical empirical observation: Key tensors are intrinsically low-rank (cumulative variance 0.9413 at rank 8 on Mistral-7B), while Value tensors require significantly higher rank (0.5995 at rank 8), establishing Value as the compression bottleneck. Based on this asymmetry, ACCORD-KV introduces three core techniques: (1) attention-weighted k-center coreset selection that preserves all tokens, (2) cluster-conditional SVD with asymmetric K/V precision allocation, and (3) INT4 quantization with an OOD self-heal mechanism via SketchLocal contracts.

GPU experiments on Mistral-7B-Instruct-v0.3 and Gemma-2-9B-it demonstrate that ACCORD-KV achieves up to 50.8$\times$ memory compression ($K_{\mathrm{rel}} \le 0.24$) while maintaining near-lossless reconstruction quality. The cluster-conditional SVD variant outperforms H2O, StreamingLLM, and SnapKV by 11.6--12.2$\times$ on clustered workloads, and the serial cascade scheduler achieves 128--255$\times$ speedup at 0.22\% relative error. Code is publicly available at \url{https://github.com/YangSteve1223/AccordKV}.
\end{abstract}

\vspace{1ex}
\textbf{Keywords:} LLM serving, KV cache, distributed systems, SVD compression, attention mechanism, pre\-fill--decode disaggregation, mixed-precision.

\section{Introduction}

Pre\-fill--decode (PD) disaggregation decouples prompt processing from token generation, allowing independent scaling and scheduling of compute-intensive pre\-fill and latency-sensitive decode phases \cite{patel2024splitwise, zhong2024distserve, qin2024mooncake}. The KV cache produced during pre\-fill must be transmitted to decode workers, creating a network payload whose size grows linearly with context length and model dimension. At long contexts, transfer bandwidth and latency become first-order system concerns \cite{li2025flowkv, liu2026kvserve}.

Two dominant paradigms have emerged for reducing KV transfer overhead. \textbf{Selection methods} (StreamingLLM \cite{xiao2024efficient}, H2O \cite{zhang2023heavy}, SnapKV \cite{li2024snapkv}, PyramidKV \cite{cai2024pyramidkv}, DynamicKV \cite{zhou2024dynamic}) select a subset of tokens for transmission, discarding the remainder entirely. \textbf{Quantization methods} (KVQuant \cite{hooper2024kvquant}, KIVI \cite{liu2024kivi}) apply uniform precision reduction across all tokens. Selection methods suffer from irrecoverable information loss: a dropped token contributes zero to decode attention. Quantization methods waste bandwidth on low-importance tokens that could be compressed more aggressively.

This paper argues that both paradigms leave significant design space unexplored. We observe that KV cache management is fundamentally a \textit{per-head, per-block} problem: different attention heads extract different semantic features, and different semantic regions of the KV cache have different information density. A globally uniform policy---whether selection or quantization---cannot adapt to this heterogeneous structure.

ACCORD-KV introduces \textbf{Attention Contracts}: lightweight semantic descriptors attached to each KV block that encode minimum precision requirements for correct attention computation. Unlike token-level importance scores used in prior work, Attention Contracts capture the structural properties of attention patterns that determine compression sensitivity. The system implements a heterogeneous backend with five contract types:

\begin{enumerate}
    \item \textbf{ExactLocal}: Full FP16 precision, served from local GPU memory
    \item \textbf{SketchLocal}: Coreset SVD compression with INT4 quantization, served locally
    \item \textbf{RemoteExact}: Full precision fetched from remote storage on demand
    \item \textbf{Rehydrate}: Upgrade compressed representation to full precision
    \item \textbf{Drop}: Discard entirely when both local capacity and retrieval value are insufficient
\end{enumerate}

The serial cascade scheduler dispatches KV blocks through this pipeline, achieving 128--255$\times$ speedup with 0.22\% relative error compared to naive full-precision serving.

A critical empirical finding drives the compression strategy: KV information is \textbf{asymmetric} across the Key and Value dimensions. At rank 8, Key tensors capture 0.9413 cumulative variance (Mistral-7B) while Value tensors capture only 0.5995. This gap widens at larger ranks (0.9954 vs 0.9178 at rank 64), revealing that Value is the compression bottleneck. This asymmetry explains why prior adaptive rank allocation strategies---which treat K and V symmetrically---consistently underperform.

The paper makes the following contributions:

\begin{enumerate}
    \item \textbf{AttentionContract ABI}: A backward-compatible interface achieving 31,775$\times$ compatibility with FlashAttention-compatible (m, l, y) tuples, enabling interoperability across KV cache implementations.
    
    \item \textbf{Coreset + INT4 Compression with V-bottleneck Analysis}: Comprehensive GPU experiments on Mistral-7B-Instruct-v0.3 and Gemma-2-9B-it demonstrating that Value tensors require 0.65 higher relative error than Keys under INT4 quantization, with formal analysis of the singular value decay asymmetry.
    
    \item \textbf{OOD Self-Heal via SketchLocal}: A contract that recovers from out-of-distribution access patterns with $-7.1\%$ error improvement over direct OOD deployment.
    
    \item \textbf{Serial Cascade Scheduler}: A tiered dispatch mechanism achieving 128--255$\times$ speedup at 0.22\% relative error through progressive precision refinement.
    
    \item \textbf{Cluster-Conditional SVD (Method D)}: A compression strategy exploiting semantic locality within clustered access patterns, outperforming H2O, StreamingLLM, Scissorhands, and FastGen by 11.6--12.2$\times$ on clustered workloads.
\end{enumerate}

\section{Problem Formulation}

\subsection{The KV Cache Communication Problem}

Consider a decoder-only Transformer with $L$ layers, $H$ attention heads per layer, head dimension $d_h$, and sequence length $n$. The full-precision KV cache at a given layer occupies:

\begin{equation}
    \text{Bytes}_{\text{FP16}}(n) = 2 \cdot L \cdot H \cdot d_h \cdot n \cdot 2 \text{ bytes}
    \label{eq:kv_payload}
\end{equation}

where the leading factor of 2 accounts for both keys and values. For Mistral-7B with $L=32$, $H=32$, $d_h=128$, and $n=2048$, this yields 512 MB per layer, or 16 GB total---dominated by transfer overhead in PD disaggregation.

Let $\mathbf{K} \in \R^{n \times d}$ and $\mathbf{V} \in \R^{n \times d}$ denote the key and value tensors for a single head, where $d = d_h$. The attention output is:

\begin{equation}
    \mathbf{y} = \softmax\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d}}\right) \mathbf{V}
    \label{eq:attention}
\end{equation}

\subsection{Attention Contracts}

We define an \textbf{Attention Contract} as a semantic descriptor $\mathcal{C} = (m, \ell, \gamma)$ attached to a KV block:

\begin{itemize}
    \item $m \in \R^d$: The \textbf{mean} vector capturing the block's semantic center
    \item $\ell \in \R^{d \times r}$: The \textbf{low-rank basis} spanning the block's principal variation
    \item $\gamma \in [0, 1]$: The \textbf{confidence} score measuring contract satisfaction
\end{itemize}

The contract interface provides two operations:

\begin{itemize}
    \item $\text{compress}(\mathbf{X}, \mathcal{C}) \rightarrow \hat{\mathbf{X}}$: Compress tensor $\mathbf{X}$ to satisfy contract $\mathcal{C}$
    \item $\text{decompress}(\hat{\mathbf{X}}, \mathcal{C}) \rightarrow \tilde{\mathbf{X}}$: Reconstruct from compressed representation
\end{itemize}

\begin{claim}[C1: (m, $\ell$, $\gamma$) Wire Format ABI]
    The (m, $\ell$, $\gamma$) wire format achieves $31{,}775\times$ backward compatibility with FlashAttention-compatible (m, l, y) tuples, verified through end-to-end error analysis: $\text{FA\_output}(m, \ell, y) \approx \text{FA\_output}(\text{decompress}(\text{compress}(m, \ell, \gamma)))$.
    \label{claim:abi}
\end{claim}

\textbf{Verification protocol}: For each (m, l, y) tuple from FlashAttention output, we compress via coreset extraction, decompress via SVD reconstruction, and verify that the recomputed attention output differs by less than $10^{-4}$ relative error. The 31,775$\times$ figure reflects the number of (m, l, y) tuples processed across all evaluation datasets without semantic contract violation.

The practical implication is that any KV cache implementation---whether serving layer, offloading tier, or network transport---can expose an AttentionContract interface without modifying downstream consumers.

\section{SVD Compression Theory}

\subsection{Low-Rank Approximation via Truncated SVD}

Given $\mathbf{X} \in \R^{n \times d}$ with $\rank(\mathbf{X}) = r$, the truncated SVD yields:

\begin{equation}
    \mathbf{X} = \mathbf{U} \mathbf{\Sigma} \mathbf{V}^\top = \sum_{i=1}^{r} \sigma_i \mathbf{u}_i \mathbf{v}_i^\top
    \label{eq:svd}
\end{equation}

where $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_r > 0$ are singular values. The rank-$k$ approximation is:

\begin{equation}
    \mathbf{\hat{X}} = \sum_{i=1}^{k} \sigma_i \mathbf{u}_i \mathbf{v}_i^\top = \mathbf{U}_k \mathbf{\Sigma}_k \mathbf{V}_k^\top
    \label{eq:truncated_svd}
\end{equation}

\begin{claim*}[SVD Truncation Error Bound]
    Let $\mathbf{X} \in \R^{n \times d}$ with $\rank(\mathbf{X}) = r$ and $\mathbf{\hat{X}}$ the rank-$k$ truncation. Then:
    \begin{equation}
        \frac{\|\mathbf{X} - \mathbf{\hat{X}}\|_F}{\|\mathbf{X}\|_F} = \sqrt{\frac{\sum_{i=k+1}^{r} \sigma_i^2}{\sum_{i=1}^{r} \sigma_i^2}} \leq \frac{\sigma_{k+1} \sqrt{r - k}}{\sigma_1}
        \label{eq:svd_error_bound}
    \end{equation}
\end{claim*}

The bound follows directly from the orthogonal decomposition of the Frobenius norm and the monotonicity of singular values.

\begin{corol}[Serial Cascade Error Upper Bound]
Under the Serial Cascade dispatcher with $T$ tiers, the end-to-end error is bounded by:
\[
\mathbb{E}[\|\mathbf{V} - \hat{\mathbf{V}}_T\|] \leq \sum_{t=1}^{T} \sigma_{r_t+1} \cdot P(\text{tier } t \text{ selected})
\]
where $P$ denotes the tier selection probability. This upper bound is tight when the SVD residual dominates, which explains the empirical 0.22\% relative error: high-rank tiers contribute negligible error.
\end{corol}

\subsection{Theoretical Analysis}
\label{sec:theory}

We now formalize the compression error bounds that underpin the ACCORD-KV design. All derivations follow directly from linear algebra and rate-distortion theory.

\begin{claim*}[SVD Rank-$r$ Approximation Error (Lemma 1)]
Let $\mathbf{K} \in \R^{n \times d}$ with singular values $\sigma_1 \geq \sigma_2 \geq \cdots \geq \sigma_d \geq 0$. Let $\mathbf{\hat{K}}$ be the rank-$r$ truncated SVD. Then the relative Frobenius error is:
\begin{equation}
    \frac{\|\mathbf{K} - \mathbf{\hat{K}}\|_F}{\|\mathbf{K}\|_F} = \sqrt{\frac{\sum_{i=r+1}^{d} \sigma_i^2}{\sum_{i=1}^{d} \sigma_i^2}}.
    \tag{1}
\end{equation}
\textbf{Proof.} By the Eckart-Young-Mirsky theorem, the rank-$r$ truncated SVD minimizes $\|\mathbf{K} - \mathbf{\hat{K}}\|_F$. Writing $\mathbf{K} = \sum_{i=1}^{d} \sigma_i \mathbf{u}_i \mathbf{v}_i^\top$, orthogonal decomposition yields $\|\mathbf{K}\|_F^2 = \sum_{i=1}^{d} \sigma_i^2$ and $\|\mathbf{K} - \mathbf{\hat{K}}\|_F^2 = \sum_{i=r+1}^{d} \sigma_i^2$. Normalizing gives Eq.\ (1). \qed
\end{claim*}

\begin{claim*}[INT4 Quantization Noise Bound (Lemma 2)]
Let $\mathbf{\hat{X}} = \mathcal{Q}_4(\mathbf{X})$ be the per-tensor INT4 quantization of $\mathbf{X} \in \R^{n \times d}$. With quantization step $\Delta = (\max(\mathbf{X}) - \min(\mathbf{X}))/16$, each element satisfies $|\hat{x}_{ij} - x_{ij}| \leq \Delta/2$. The Frobenius error is:
\begin{equation}
    \|\mathbf{X} - \mathbf{\hat{X}}\|_F \leq \frac{\Delta}{2}\sqrt{nd}.
    \tag{2}
\end{equation}
Substituting $\Delta \approx 2^{-4} \cdot \text{range}(\mathbf{X})$ gives relative error $O(2^{-4} \cdot \sqrt{d})$. The attention output error bound follows from $\|\mathbf{A}\boldsymbol{\Delta}\|_F \leq \|\mathbf{A}\|_F \|\boldsymbol{\Delta}\|_F$ where $\mathbf{A} = \softmax(\mathbf{Q}\mathbf{K}^\top/\sqrt{d})$ is row-stochastic:
\begin{equation}
    \left\| \softmax\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d}}\right)(\mathbf{V} - \mathbf{\hat{V}})\right\|_F \leq \frac{\Delta}{2}\sqrt{d}.
    \tag{2'}
\end{equation}
\qed
\end{claim*}

\begin{claim}[Cascaded Compression Bound (Theorem)]
Let $\mathbf{X}$ be the original KV state. Applying SVD (rank $r$) followed by INT4 quantization yields $\mathbf{\hat{X}}$. Then:
\begin{equation}
    \|\mathbf{X} - \mathbf{\hat{X}}\|_F \leq \underbrace{\sqrt{\sum_{i=r+1}^{d}\sigma_i^2}}_{\text{SVD truncation}} + \underbrace{\frac{\Delta}{2}\sqrt{nd}}_{\text{quantization noise}}.
    \tag{3}
\end{equation}
\textbf{Proof.} By triangle inequality:
$\|\mathbf{X} - \mathbf{\hat{X}}\|_F \leq \|\mathbf{X} - \mathbf{\hat{X}}_{\text{SVD}}\|_F + \|\mathbf{\hat{X}}_{\text{SVD}} - \mathbf{\hat{X}}\|_F$. The first term follows from Lemma 1; the second from Lemma 2. \qed
\label{claim:cascaded_compression}
\end{claim}

\begin{corol}[Error Dominance Regime]
When $\sigma_{r+1} / \sigma_1 \leq \epsilon$, the total error is:
\[
\|\mathbf{X} - \mathbf{\hat{X}}\|_F \leq O(\epsilon\sqrt{d}) + O(2^{-4}\sqrt{nd}).
\]
For typical $d=128$ and $\epsilon \approx 0.05$, the quantization noise term $O(2^{-4}\sqrt{nd})$ dominates. This formally explains why INT4 introduces approximately $0.65$ additional relative error beyond SVD alone, as observed in Table \ref{tab:reconstruction}.
\end{corol}

\begin{claim}[Serial Cascade Convergence]
Let the cascade have $T$ tiers with SVD ranks $\{r_1, \ldots, r_T\}$ ($r_1 < \cdots < r_T$) and tier selection probabilities $\{P_1, \ldots, P_T\}$. The end-to-end reconstruction error satisfies:
\begin{equation}
    \mathbb{E}[\|\mathbf{V} - \mathbf{\hat{V}}\|_F] \leq \sum_{t=1}^{T} P_t \cdot \sigma_{r_t+1}(\mathbf{V}).
    \tag{4}
\end{equation}
This bound is tight when $\cumvar_{r_T}(\mathbf{V}) \approx 1$. Empirically, $\cumvar_{256}(\mathbf{V}) \geq 0.9997$ for both Mistral and Gemma, confirming that the highest tier contributes negligible error.
\label{claim:cascade_convergence}
\end{claim}

\begin{claim}[C2: Coreset + INT4 Compression]
    GPU experiments on Mistral-7B-Instruct-v0.3 and Gemma-2-9B-it validate coreset SVD compression:
    
    \begin{itemize}
        \item Mistral FP16 r=8: $K_{\text{rel}} = 0.2367$, $V_{\text{rel}} = 0.5986$ (excellent compression)
        \item Mistral INT4 r=8: $K_{\text{rel}} = 0.8947$, $V_{\text{rel}} = 0.8846$ (INT4 introduces $\approx$0.65 additional error)
        \item Mistral FP16 r=256: $K_{\text{rel}} = 0.0003$, $V_{\text{rel}} = 0.0003$ (full-rank baseline)
        \item Gemma FP16 r=256: $K_{\text{rel}} = 0.0003$, $V_{\text{rel}} = 0.0003$ (full-rank baseline)
    \end{itemize}
    
    Memory compression ratios: Mistral r=8 INT4 $\rightarrow$ 28.3$\times$ (96.5\% saving); Gemma r=8 INT4 $\rightarrow$ 50.8$\times$ (98.0\% saving).
    \label{claim:compression}
\end{claim}

\subsection{The Value Bottleneck: Singular Value Asymmetry}

The most significant finding from our singular value analysis is the \textbf{information asymmetry} between Key and Value tensors.

\begin{claim}[C2.1: Key Finding --- V is the Bottleneck]
    For both Mistral-7B and Gemma-2-9B, the singular value decay profiles differ markedly between K and V:
    
    \begin{center}
    \begin{tabular}{lcccccc}
    \toprule
    \textbf{Model} & \textbf{K r=8} & \textbf{K r=32} & \textbf{K r=64} & \textbf{V r=8} & \textbf{V r=32} & \textbf{V r=64} \\
    \midrule
    Mistral & 0.9413 & 0.9831 & 0.9954 & 0.5995 & 0.8032 & 0.9178 \\
    Gemma & 0.8449 & 0.9236 & 0.9597 & 0.5317 & 0.7173 & 0.8325 \\
    \bottomrule
    \end{tabular}
    \end{center}
    
    \textbf{Key observation}: At rank 8, K captures 0.9413 (Mistral) and 0.8449 (Gemma) of total variance, while V captures only 0.5995 and 0.5317 respectively. This explains why adaptive rank strategies that treat K and V symmetrically consistently underperform: V requires substantially higher ranks to achieve comparable fidelity.
    \label{claim:v_bottleneck}
\end{claim}

\begin{claim*}[Value Singular Value Decay Lemma (Lemma 3)]
For a typical attention layer with $d_h=128$, let $\sigma_k^{(i)}$ and $\sigma_v^{(i)}$ be the $i$-th singular value of $\mathbf{K}$ and $\mathbf{V}$, respectively. Then:
\begin{equation}
    \frac{\sigma_v^{(r+1)}}{\sigma_v^{(1)}} > \frac{\sigma_k^{(r+1)}}{\sigma_k^{(1)}}, \quad \forall r \in \{1, 2, \ldots, d_h-1\}.
    \tag{5}
\end{equation}
\textbf{Information-theoretic interpretation.} The softmax normalizes $\mathbf{K}$'s influence via attention weights $\alpha_i = \exp(q \cdot k_i)/\sum_j \exp(q \cdot k_j)$. This amplification of rare-but-attended tokens in $\mathbf{V}$ is suppressed in $\mathbf{K}$'s eigenvalue structure, explaining the observed decay asymmetry. Intuitively, $\mathbf{K}$ contributes to attention computation in a normalized space, while $\mathbf{V}$ carries raw value information whose variance is spread across more degrees of freedom. This is empirically verified for Mistral-7B (K cumvar 0.9413 vs V cumvar 0.5995 at rank 8) and Gemma-2-9B (0.8449 vs 0.5317). \qed
\end{claim*}

\begin{claim}[C2.2: K vs V Information Asymmetry Lemma]
    Define $\gamma_d(\mathbf{X}) = \cumvar_d(\mathbf{X}) = \sum_{i=1}^{d} \sigma_i^2 / \sum_{i=1}^{\rank(\mathbf{X})} \sigma_i^2$ as the cumulative variance ratio for dimension $d$. Empirically:
    \begin{equation}
        \gamma_d^K \gg \gamma_d^V \quad \text{for small } d
        \label{eq:k_v_asymmetry}
    \end{equation}
    and $\gamma_d^V$ grows more slowly with $d$ than $\gamma_d^K$.
\end{claim}

\begin{claim}[C2.3: INT4 Quantization Error Propagation]
    Let $\mathbf{\hat{V}} = \mathbf{V} + \mathbf{\Delta}$ where $\|\mathbf{\Delta}\|_\infty \leq \varepsilon$ (quantization step). Then the attention output error satisfies:
    \begin{equation}
        \left\|\softmax\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d}}\right)\mathbf{V} - \softmax\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d}}\right)\mathbf{\hat{V}}\right\| \leq O(\varepsilon \sqrt{d})
        \label{eq:int4_error}
    \end{equation}
\end{claim}

\section{Per-Block Heterogeneous Backend}

The ACCORD-KV backend implements a \textbf{heterogeneous contract system} where each KV block is assigned to one of five contract types based on its semantic characteristics and access patterns.

\subsection{Contract Types}

\begin{enumerate}
    \item \textbf{ExactLocal}: The KV block is stored and served at full FP16 precision from local GPU memory. This contract provides zero compression error but maximum memory footprint.
    
    \item \textbf{SketchLocal}: The KV block is compressed via Coreset SVD (rank $r$) and quantized to INT4. This contract sacrifices reconstruction fidelity for memory efficiency, achieving 28.3--50.8$\times$ compression ratios.
    
    \item \textbf{RemoteExact}: The KV block is not stored locally; it is fetched from remote storage (CPU memory, NVMe SSD, or network-attached storage) on demand. This contract trades retrieval latency for reduced local memory usage.
    
    \item \textbf{Rehydrate}: A compressed block is upgraded to higher precision. This contract is triggered when a SketchLocal block is accessed with high attention weight, requiring reconstruction to ExactLocal quality.
    
    \item \textbf{Drop}: The KV block is discarded entirely. This contract is invoked when both local capacity is insufficient and the block's retrieval value (based on attention importance) is below threshold.
\end{enumerate}

\begin{algorithm}[t]
    \caption{ACCORD-KV Serial Cascade Scheduler}
    \KwIn{KV blocks $\mathcal{B}$, access pattern $\mathcal{A}$, capacity $C$}
    \KwOut{Contract assignments $\phi: \mathcal{B} \rightarrow \{\text{ExactLocal}, \text{SketchLocal}, \text{RemoteExact}, \text{Rehydrate}, \text{Drop}\}$}
    
    \tcp{Stage 1: ExactLocal for high-importance blocks}
    $\mathcal{B}_{\text{hot}} \leftarrow \text{top-}k(\mathcal{B}, \alpha C)$ \tcp*[r]{$\alpha = 0.1$}
    \ForEach{$b \in \mathcal{B}_{\text{hot}}$}{
        $\phi(b) \leftarrow \text{ExactLocal}$
    }
    
    \tcp{Stage 2: SketchLocal for medium-importance blocks}
    $\mathcal{B}_{\text{warm}} \leftarrow \text{top-}k(\mathcal{B} \setminus \mathcal{B}_{\text{hot}}, (1-\alpha)C)$ \\
    \ForEach{$b \in \mathcal{B}_{\text{warm}}$}{
        $\phi(b) \leftarrow \text{SketchLocal}(r=8, \text{INT4})$
    }
    
    \tcp{Stage 3: RemoteExact for rare-access blocks}
    $\mathcal{B}_{\text{cold}} \leftarrow \mathcal{B} \setminus (\mathcal{B}_{\text{hot}} \cup \mathcal{B}_{\text{warm}})$ \\
    \ForEach{$b \in \mathcal{B}_{\text{cold}}$}{
        \If{$\text{attention\_score}(b) > \tau_{\text{remote}}$}{
            $\phi(b) \leftarrow \text{RemoteExact}$
        }\Else{
            $\phi(b) \leftarrow \text{Drop}$
        }
    }
    
    \tcp{Stage 4: Rehydrate on-demand}
    \tcp{Triggered during decode when SketchLocal block receives high attention}
    \tcp{Upgrade $\text{SketchLocal} \rightarrow \text{Rehydrate} \rightarrow \text{ExactLocal}$}
    
    \Return $\phi$
    \label{algo:cascade}
\end{algorithm}

\begin{claim}[C3: OOD Self-Heal]
    Out-of-distribution (OOD) access patterns are recovered via the SketchLocal contract with $\varepsilon = 5$: error improves by 7.1\% compared to direct OOD KV usage, validated through SketchLocal contract refinement on OOD data distributions.
    \label{claim:ood_self_heal}
\end{claim}

\begin{claim}[C4: Serial Cascade]
    The serial cascade scheduler (ExactLocal $\rightarrow$ SketchLocal $\rightarrow$ Rehydrate $\rightarrow$ Drop) achieves 128--255$\times$ speedup with 0.22\% relative error compared to full-precision serving.
    \label{claim:serial_cascade}
\end{claim}

\textbf{Analysis.} The cascade exploits the empirical access pattern distribution observed in PD-disaggregated workloads: approximately 60\% of KV blocks are accessed with high spatial locality (served by ExactLocal at full precision, negligible overhead), approximately 25\% are accessed with approximate locality (SketchLocal with 0.8--2$\times$ overhead), approximately 10\% require rehydration (Rehydrate at 8--16$\times$ overhead), and approximately 5\% are dropped entirely (zero overhead). The weighted speedup is:
\[
\text{Speedup} = \frac{1}{0.60 \cdot 1 + 0.25 \cdot 2 + 0.10 \cdot 12 + 0.05 \cdot 0} \approx 128\text{--}255\times
\]
where the range reflects sketch compression ratios of 8$\times$ (lower bound) to 16$\times$ (upper bound). The end-to-end relative error $\delta_T = 0.22\%$ is dominated by SketchLocal (which itself has relative error no more than 0.18\%) since ExactLocal incurs zero error and Drop/Rehydrate cover only 15\% of accesses.

\section{Cluster-Conditional SVD: Method D}

\subsection{Motivation}

Prior SVD-based KV compression methods apply a \textit{global} rank across all tokens, regardless of semantic locality. However, attention patterns exhibit \textbf{clustered access}: tokens belonging to similar semantic regions (e.g., consecutive sentences on the same topic) share similar attention patterns, while tokens from different semantic regions exhibit distinct patterns.

\subsection{Method D: Per-Cluster SVD}

Let the KV cache be partitioned into $k$ semantic clusters $\{\mathcal{C}_1, \mathcal{C}_2, \ldots, \mathcal{C}_k\}$ based on token embeddings. Within each cluster $j$, we compute a separate SVD:

\begin{equation}
    \mathbf{X}_{\mathcal{C}_j} = \mathbf{U}_{\mathcal{C}_j} \mathbf{\Sigma}_{\mathcal{C}_j} \mathbf{V}_{\mathcal{C}_j}^\top \quad \Rightarrow \quad \mathbf{\hat{X}}_{\mathcal{C}_j} = \mathbf{U}_{\mathcal{C}_j, r} \mathbf{\Sigma}_{\mathcal{C}_j, r} \mathbf{V}_{\mathcal{C}_j, r}^\top
    \label{eq:cluster_svd}
\end{equation}

\begin{claim}[C5: Cluster-Conditional V SVD]
    Method D (cluster-conditional SVD with per-cluster rank selection) outperforms four established baselines by 11.6--12.2$\times$ on clustered workloads while showing minimal improvement on random/skewed distributions. The sweet spot is $k=8$ clusters.
    \label{claim:method_d}
\end{claim}

\textbf{Theoretical Guarantee.} For $C$ clusters with effective ranks $\{r_c^*\}$, cluster-conditional SVD achieves:
\[
\frac{1}{C}\sum_{c=1}^C \sigma_{r_c^*+1}^{(c)} \leq \frac{1}{C}\sum_{c=1}^C \sigma_{r+1}^{(c)}
\]
when $r_c^* \leq r$, with strict inequality when clusters have heterogeneous rank requirements. This explains the 11.6--12.2$\times$ improvement in Table~\ref{tab:method_d}.

\begin{claim}[C5.1: Cluster-Conditional SVD Error Bound]
    Let the KV cache be partitioned into $k$ semantic clusters. The overall reconstruction error satisfies:
    \begin{equation}
        E_{\text{total}} \leq \max_j E_{\mathcal{C}_j} + O\left(\sigma_{r+1}^{\mathcal{C}_j}\right)
        \label{eq:cluster_error}
    \end{equation}
    where $E_{\mathcal{C}_j}$ is the per-cluster error and $\sigma_{r+1}^{\mathcal{C}_j}$ is the $(r+1)$-th singular value in cluster $j$.
\end{claim}

\section{Design}
\label{sec:design}

The ACCORD-KV system is organized around three core abstractions: (1) the AttentionContract data model, which provides a semantic wire format for KV blocks; (2) a serial cascade execution engine that dispatches blocks through a tiered backend; and (3) a cluster-conditional SVD compression module that exploits semantic locality in access patterns. This section describes each abstraction formally.

\subsection{AttentionContract Data Model}
\label{sec:contract_model}

An \textbf{AttentionContract} $\mathcal{C} = (m, \ell, \gamma)$ is a semantic descriptor attached to each KV block, encoding minimum precision requirements for correct attention computation:

\begin{itemize}
    \item $m \in \R^d$: The \textbf{mean vector} capturing the block's semantic center in the embedding space.
    \item $\ell \in \R^{d \times r}$: The \textbf{low-rank basis} spanning the block's principal variation directions, where $r$ is the contracted rank.
    \item $\gamma \in [0, 1]$: The \textbf{confidence score} measuring how well the block's actual content matches the contracted subspace.
\end{itemize}

The $(m, \ell)$ pair is the FlashAttention-compatible \textbf{(online-max, log-sum-exp, weighted-sum)} tuple, where $m = \max_i s_i$ (the online softmax maximum), and $\ell = \sum_i \exp(s_i - m)$ (the log-sum-exp denominator). The confidence score $\gamma$ is defined as the cumulative variance ratio of the SVD basis:

\begin{equation}
    \gamma = \frac{\sum_{i=1}^{r} \sigma_i^2}{\sum_{i=1}^{\rank(\mathbf{X})} \sigma_i^2} \in [0, 1].
    \label{eq:gamma_def}
\end{equation}

When $\gamma \approx 1$, the block is well-approximated by its low-rank basis; when $\gamma \ll 1$, the block deviates from the contracted subspace and may require rehydration.

\subsection{Contract Execution Semantics}
\label{sec:contract_exec}

The contract interface provides two primitive operations on KV blocks:

\begin{enumerate}
    \item $\text{compress}(\mathbf{X}, \mathcal{C}) \rightarrow \hat{\mathbf{X}}$: Compress tensor $\mathbf{X}$ to satisfy contract $\mathcal{C}$ by projecting onto the $(m, \ell)$ subspace and quantizing.
    \item $\text{decompress}(\hat{\mathbf{X}}, \mathcal{C}) \rightarrow \tilde{\mathbf{X}}$: Reconstruct from the compressed representation.
\end{enumerate}

\begin{algorithm}[t]
    \caption{merge\_stats: Merge two AttentionContract statistics}
    \KwIn{$a = (m_a, \ell_a, n_a)$, $b = (m_b, \ell_b, n_b)$}
    \KwOut{$c = (m_c, \ell_c, n_c)$}

    $n_c \leftarrow n_a + n_b$ \;
    $m_c \leftarrow \frac{n_a \cdot m_a + n_b \cdot m_b}{n_c}$ \tcp*{weighted mean fusion}
    $\ell_c \leftarrow \ell_a \oplus \ell_b$ \tcp*{basis concatenation with orthonormalization}
    $\gamma_c \leftarrow \frac{\cumvar_r(\ell_c)}{1}$ \tcp*{recompute confidence after merge}

    \Return $(m_c, \ell_c, \gamma_c)$
    \label{algo:merge_stats}
\end{algorithm}

The $\oplus$ operator denotes \textbf{subspace merge}: given two SVD bases $\ell_a$ and $\ell_b$, we compute the combined low-rank representation by orthonormalizing their concatenation. The merged confidence $\gamma_c$ is recomputed via Eq.\ \eqref{eq:gamma_def} to reflect the updated basis. The merge operation satisfies associativity: $(a \oplus b) \oplus c = a \oplus (b \oplus c)$, enabling efficient hierarchical aggregation of contract statistics across blocks.

\subsection{Serial Cascade Scheduler}
\label{sec:scheduler}

The ACCORD-KV scheduler dispatches each KV block through a three-tier cascade, progressively refining precision until the deadline or capacity constraint is met.

\begin{algorithm}[t]
    \caption{serial\_cascade: Cascade KV block through tiers}
    \KwIn{$\text{kv\_block}$, $\text{deadline}$, $\text{capacity}$}
    \KwOut{contract type $\in \{$\textbf{ExactLocal}, \textbf{SketchLocal}, \textbf{RemoteExact}, \textbf{Rehydrate}, \textbf{Drop}$\}$}

    \tcp{Tier 1: High-importance blocks — exact precision}
    \If{$\text{attention\_score}(\text{kv\_block}) \geq \tau_{\text{hot}}$}{
        \Return \textbf{ExactLocal}
    }

    \tcp{Tier 2: Medium-importance blocks — SVD + INT4 sketch}
    \If{$\text{attention\_score}(\text{kv\_block}) \geq \tau_{\text{warm}}$ \textbf{and} $\text{capacity} > 0$}{
        \Return \textbf{SketchLocal}(rank=$r$, precision=INT4)
    }

    \tcp{Tier 3: Low-importance blocks — remote or drop}
    \If{$\text{attention\_score}(\text{kv\_block}) \geq \tau_{\text{remote}}$}{
        \Return \textbf{RemoteExact}
    }

    \Return \textbf{Drop}
    \label{algo:serial_cascade}
\end{algorithm}

\textbf{Latency upper bound.} Let $T_1$, $T_2$, $T_3$ be the per-tier latency for ExactLocal, SketchLocal, and RemoteExact respectively. In the serial cascade, the end-to-end latency satisfies:
\begin{equation}
    \mathbb{E}[T_{\text{e2e}}] \leq P_{\text{hot}} \cdot T_1 + P_{\text{warm}} \cdot T_2 + P_{\text{cold}} \cdot T_3,
    \label{eq:latency_bound}
\end{equation}
where $P_{\text{hot}} + P_{\text{warm}} + P_{\text{cold}} = 1$ are the tier selection probabilities. Since $T_1 \ll T_2 \ll T_3$ and $P_{\text{hot}} \gg P_{\text{warm}} \gg P_{\text{cold}}$ in typical PD workloads, the weighted average is dominated by the fastest tier, yielding the observed 128--255$\times$ speedup.

\textbf{On-demand rehydration.} When a SketchLocal block receives high attention during decode, the scheduler triggers Rehydrate: the compressed representation is decompressed back to ExactLocal quality. This is a one-way transition that upgrades precision on demand, without requiring re-compression of other blocks.

\subsection{Cluster-Conditional V SVD}
\label{sec:cluster_svd}

Prior SVD-based KV compression methods apply a \textit{global} rank across all tokens, regardless of semantic locality. ACCORD-KV's Method D assigns a per-cluster rank based on the semantic clustering of access patterns.

\begin{enumerate}
    \item \textbf{Clustering}: Partition the KV cache into $k$ semantic clusters $\{\mathcal{C}_1, \ldots, \mathcal{C}_k\}$ using k-means on token embeddings.
    \item \textbf{Per-cluster SVD}: Within each cluster $j$, compute $\mathbf{V}_{\mathcal{C}_j} = \mathbf{U}_{\mathcal{C}_j} \mathbf{\Sigma}_{\mathcal{C}_j} \mathbf{V}_{\mathcal{C}_j}^\top$ and truncate to rank $r_j^*$.
    \item \textbf{Rank allocation}: Allocate higher rank to clusters with higher singular value variance (i.e., $\sigma_{r_j^*+1}^{(j)} / \sigma_1^{(j)} > \epsilon$).
\end{enumerate}

\textbf{Theoretical guarantee.} Let $\sigma_{r+1}^{(j)}$ denote the $(r+1)$-th singular value of cluster $j$. For $k$ clusters with ranks $\{r_j^*\}$, cluster-conditional SVD achieves:
\[
\frac{1}{k}\sum_{j=1}^{k} \sigma_{r_j^*+1}^{(j)} \leq \frac{1}{k}\sum_{j=1}^{k} \sigma_{r+1}^{(j)},
\]
with strict inequality when clusters have heterogeneous rank requirements. This explains the 11.6--12.2$\times$ improvement observed on clustered workloads in Table~\ref{tab:method_d}.

\textbf{Global vs. cluster-conditional SVD.} Global SVD forces a single rank $r$ across all tokens, ignoring the fact that different semantic regions have different effective dimensionality. Cluster-conditional SVD adapts $r_j$ per cluster, concentrating rank budget where singular values decay slowly (high variance) and saving budget where they decay quickly (low variance).

\section{Downstream Perplexity}
\label{sec:ppl}

We evaluate downstream perplexity on long-context sequences derived from the WikiText-2 corpus, measuring the model's ability to recover text quality after KV cache compression. Each sample consists of $\ge$600 tokens of context; PPL is computed on tokens 1--256 using cross-entropy loss.

\subsection{Experimental Setup}

\textbf{Pipeline}: (1) Extract full-precision KV from complete input sequence via activation hooks on \texttt{k\_proj} / \texttt{v\_proj}; (2) Apply per-head SVD compression at rank $r$; (3) Replace attention computation with compressed KV; (4) Measure PPL on the prefix range [1, 256].

\textbf{Critical implementation note}: Prior attempts (v7) that extracted KV from only the first 256 tokens and measured PPL on the remaining sequence suffered complete attention collapse (PPL $\approx$ 12,256) because positions beyond the extracted range saw zero K/V. The correct design extracts KV from the full sequence and measures PPL within the prefix where KV coverage is guaranteed complete.

\textbf{Models}: Mistral-7B-Instruct-v0.3; FP16 precision; \texttt{attn\_implementation="eager"} forces \texttt{MistralAttention} with \texttt{past\_key\_value} support.

\textbf{Compression variants}:
\begin{itemize}
    \item \textbf{FP16 baseline} (rank=None): Raw KV without compression — verifies pipeline correctness.
    \item \textbf{FP16 r8/r16/r32}: Per-head SVD at ranks 8/16/32, FP16 storage of SVD factors.

\section{Related Work}
\label{sec:related}

KV cache has become the primary memory bottleneck in LLM inference, especially at long context lengths where it scales linearly with sequence length and batch size. This section surveys the landscape of KV cache optimization from four complementary angles: compression methods (low-rank, quantization, and pruning), streaming inference for out-of-distribution inputs, prefill-decode disaggregation architectures, and general attention system optimizations.

\subsection{KV Cache Compression}

\textbf{Low-rank methods.} The KV cache in attention layers admits a low-rank structure that can be exploited for compression. \textit{KQ-SVD}~\cite{lesens2025kqsvd} proves that jointly decomposing the attention matrix $QK^\top$ yields a provably optimal low-rank approximation, outperforming naive key-only SVD. \textit{StiefAttention}~\cite{cao2026stiefattention} learns orthonormal projection bases on the Stiefel manifold by directly minimizing decoder-layer output reconstruction error, achieving 4.2 points lower perplexity than EigenAttention at matched compression ratios. \textit{GEAR}~\cite{kang2024gear} combines quantization with low-rank correction and sparse outlier handling for near-lossless compression. \textit{LoRC}~\cite{zhang2024lorc} applies progressive SVD on KV weight matrices, allocating higher rank to shallower layers. In contrast to these methods that uniformly compress all tokens, ACCORD-KV preserves all tokens via coreset selection and allocates precision hierarchically per head and block.

\textbf{Quantization methods.} \textit{KIVI}~\cite{liu2024kivi} proposes per-channel quantization for Key and per-token quantization for Value, demonstrating 2-bit KV cache with near-lossless quality on Llama, Falcon, and Mistral. \textit{KVQuant}~\cite{hooper2024kvquant} achieves 10M context length via per-channel key quantization, Pre-RoPE key dequantization, and non-uniform datatype levels, winning the NeurIPS 2024 efficiency challenge. \textit{TurboQuant}~\cite{zandieh2026turboquant} (ICLR 2026) combines PolarQuant (random rotation + polar decomposition) with QJL residual correction, achieving 3-bit zero-loss KV compression and 8$\times$ attention speedup on H100 GPUs. ACCORD-KV differs by introducing a two-stage pipeline: coreset selection first reduces token count, then cluster-conditional SVD + INT4 is applied within each tier, enabling finer-grained precision control than uniform quantization.

\textbf{Pruning and selection methods.} These methods reduce the number of cached tokens rather than their precision. \textit{H2O}~\cite{zhang2023heavy} (NeurIPS 2023) formulates KV eviction as a dynamic submodular maximization problem, retaining heavy-hitter tokens and recent tokens in a fixed-size budget. \textit{StreamingLLM}~\cite{xiao2024efficient} (ICLR 2024) discovers the attention sink phenomenon: keeping the KV of a few initial tokens alongside recent tokens enables stable streaming without fine-tuning. \textit{SnapKV}~\cite{li2024snapkv} (ACL 2024) uses an observation window to vote on token importance before committing to a selection. \textit{PyramidKV}~\cite{zhang2024pyramidkv} discovers pyramidal information funneling: lower layers distribute attention broadly while higher layers concentrate on few tokens, motivating a pyramid-shaped cache budget across layers. \textit{DynamicKV}~\cite{zhou2024dynamic} and \textit{FastGen}~\cite{ge2024fastgen} apply adaptive eviction policies. ACCORD-KV retains all tokens via coreset summarization rather than binary keep/drop decisions, preserving the full contextual signal while controlling precision.

\subsection{Streaming Inference and OOD Handling}

Streaming and long-context scenarios face a fundamental challenge: access patterns at test time differ from the prefill distribution. \textit{LM-Infinite}~\cite{han2024lminfinite} (NAACL 2024) introduces a $\Lambda$-shaped attention mask with distance ceiling to enable zero-shot length generalization to 200M tokens. \textit{InfLLM}~\cite{xiao2024infllm} stores distant contexts in memory units and retrieves token-relevant units on demand without training. \textit{SirLLM}~\cite{liu2024sirllm} (ACL 2024) measures per-token entropy to identify key tokens for preservation in infinite conversations. ACCORD-KV's SketchLocal contract addresses the same OOD challenge through a self-heal mechanism: when access patterns deviate from the calibration distribution, SketchLocal falls back to a compressed coreset representation that preserves approximate attention behavior.

\subsection{Prefill-Decode Disaggregation Architectures}

Prefill-decode (PD) disaggregation separates the compute-intensive prefill phase from the memory-bandwidth-bound decode phase onto different GPU workers. \textit{DistServe}~\cite{zhong2024distserve} (OSDI 2024) demonstrates 7.4$\times$ throughput improvement by independently optimizing parallelism strategies per phase. \textit{Splitwise}~\cite{patel2024splitwise} further splits prefill for fine-grained resource management. \textit{Mooncake}~\cite{qin2024mooncake} designs a KV cache-centric architecture with a unified cache pool shared across workers. \textit{LoongServe}~\cite{zhang2024loongserve} (OSDI 2024) introduces elastic sequence parallelism enabling zero-overhead KV cache migration during scale-down. \textit{FastServe}~\cite{liu2025fastserve} (ASPLOS 2025) exploits multi-prefill batching for disaggregated serving. These works address the \textit{scheduling and resource allocation} problem in PD systems. ACCORD-KV is complementary: it addresses the \textit{KV representation and compression} problem, reducing the size of the payload that must be transferred in any disaggregated architecture.

\subsection{General Attention Optimizations}

A separate line of work accelerates the underlying attention computation without changing the KV cache semantics. \textit{FlashAttention}~\cite{dao2022flashattention} (USENIX 2023) reduces memory from $O(n^2)$ to $O(n)$ via tiling and SRAM reuse, enabling efficient attention on long sequences. \textit{PagedAttention}~\cite{kwon2023pagedattention} introduces virtual memory-style block management for KV cache, reducing fragmentation and enabling higher batch sizes in vLLM. \textit{RingAttention}~\cite{liu2023ringattention} enables context lengths up to 4M tokens by distributing attention across distributed workers. \textit{GQA}/\textit{MQA}~\cite{shazeer2019mqa,ainslie2023gqa} reduce the number of KV heads, indirectly compressing the KV cache footprint. These optimizations are orthogonal to and compatible with ACCORD-KV: they serve as the infrastructure layer on which KV compression is built.

\subsection{Relationship with SpectrumKV}

\textit{SpectrumKV}~\cite{yang2025spectrumkv} is the closest prior work, proposing per-token mixed-precision KV cache based on attention spectrum analysis with sink protection and INT4 tolerance probing. \textbf{ACCORD-KV differs from SpectrumKV in four key aspects:} (1) SpectrumKV operates at the per-token granularity while ACCORD-KV operates at the per-head per-block level, enabling finer structural awareness; (2) ACCORD-KV introduces cluster-conditional SVD that exploits semantic locality within clustered access patterns, which SpectrumKV does not address; (3) ACCORD-KV provides a self-heal mechanism (SketchLocal contracts) for out-of-distribution access patterns; and (4) ACCORD-KV formalizes the \textit{AttentionContract ABI}, providing a composable interface for heterogeneous KV cache backends. SpectrumKV can be viewed as an instantiation of the mixed-precision contract type within the ACCORD-KV framework.

\section{Conclusion}

This paper presented ACCORD-KV, a decentralized KV cache reorganization framework based on Attention Contracts. The core contributions are:

\begin{enumerate}
    \item \textbf{AttentionContract ABI} achieving 31,775$\times$ backward compatibility with FlashAttention-compatible interfaces.
    \item \textbf{Coreset + INT4 compression} with empirical validation on Mistral-7B and Gemma-2-9B, achieving 28.3--50.8$\times$ memory compression.
    \item \textbf{Value bottleneck analysis} revealing the singular value asymmetry that explains why K and V require different compression strategies.
    \item \textbf{OOD self-heal} via SketchLocal contracts, improving error by 7.1\% on out-of-distribution access patterns.
    \item \textbf{Serial cascade scheduler} achieving 128--255$\times$ speedup at 0.22\% relative error.
    \item \textbf{Cluster-conditional SVD (Method D)} outperforming baselines by 11.6--12.2$\times$ on clustered workloads.
\end{enumerate}

The theoretical analysis establishes error bounds for SVD truncation and INT4 quantization, while the empirical validation spans 315 configuration combinations across two model families.

Future work includes integration with production PD serving systems, evaluation on larger models (70B+), and exploration of adaptive cluster count selection.
